# EnglishLover Web Foundation

本目录是 EnglishLover Web 端第一轮基础骨架。当前目标是“仅搭框架、不做业务页、不接真实接口、不引入临时数据”。

## 可运行命令

```bash
npm install
npm run dev
npm run typecheck
npm run lint
npm run build
```

## 目录职责

| 目录                   | 职责                              | 第一轮边界                                      |
| ---------------------- | --------------------------------- | ----------------------------------------------- |
| `src/pages/`           | 路由壳、占位页、组件示例页        | 不写依赖接口的业务页                            |
| `src/features/`        | 后续业务状态编排和真实接口调用    | 当前仅保留边界说明                              |
| `src/components/`      | 跨业务基础组件与布局组件          | 不请求接口，不生成业务数据                      |
| `src/types/`           | 类型入口                          | `api.ts` 只能由后端契约自动生成；当前为占位说明 |
| `src/lib/`             | 路由、工具函数等基础能力          | 不放业务规则                                    |
| `src/styles/`          | 设计令牌、Tailwind 映射、全局样式 | 页面不得硬编码颜色和任意样式                    |
| `src/docs/ai-context/` | 后续 AI Studio 上下文摘要         | 当前只记录边界和占位规则                        |

## 已建立路由

- `/`：框架总览；
- `/today`、`/vocabulary`、`/reading`、`/penpal`、`/results`：空壳路由；
- `/components`：P0 基础组件示例页。

所有业务范围内路由只展示“壳已建好、业务未开始”的状态，不展示列表、表格、学习记录、用户信息、课程内容或任何临时数据。

## 后端协作约束

前端不得提前消费未冻结契约。后端应并行推进冻结前评审准备：统一接口命名、错误码、枚举字典、分页结构、认证方式和响应包裹格式，并开始接口编码与契约测试用例编写。待契约冻结后，再自动生成并冻结 `src/types/api.ts`。
