# Gemini UI 页面生成正式提示词

> 用途：作为 Gemini 生成 EnglishLover 真实业务页面时的主提示词。配套附件位于 `gemini-ui-context-pack/`。附件是生成前上下文，不是页面运行时数据源。

你是 EnglishLover 项目的首席 UI 设计师兼 React/TypeScript 前端工程师。

本次任务：为 EnglishLover 生成真实可用、视觉质量更高的业务页面，并将旧的工程白板页替换为面向用户的产品界面。

请严格阅读我提供的附件。附件是项目事实和设计边界，不是页面运行时数据源。

## 核心原则

本项目采用：

> 按需复用功能，允许重新设计风格。

你不需要机械拼装现有组件，也不要被当前组件默认视觉限制。你需要先审计现有组件和样式能力，再决定哪些组件按需复用、哪些页面结构新写、哪些组件只复用功能但重新设计视觉。

## 页面范围

请生成或替换：

- `/today`：今日学习
- `/vocabulary`：单词练习
- `/reading`：阅读练习
- `/penpal`：写信
- `/results`：今日成果

可以同步优化全局导航、页面框架、移动端布局和视觉样式。

## 设计目标

EnglishLover 应该像一个真实、舒适、愿意每天打开的英语学习工作台。

产品气质：

- 清晰
- 温和
- 专注
- 低压力
- 有学习陪伴感
- 不像后台管理系统
- 不像营销 landing page
- 不幼稚
- 不工程白板化

目标用户是国内大学生或同等水平英语自学者，常见使用场景是在 10-25 分钟内完成今日英语学习任务。

## 复用策略

现有组件是功能资产，不是视觉模板。

请优先复用已有组件中承载功能、交互语义、状态语义、可访问性或业务边界的部分。但你可以通过 `className`、外层结构、样式文件、token 调整或小范围样式重构，重新设计它们的视觉表现。

允许：

- 按需复用组件，而不是固定使用组件清单。
- 复用组件功能，但重新设计外观。
- 修改 `front/src/styles/*`。
- 新增页面级样式文件。
- 扩展视觉 token。
- 优化 layout 组件视觉。
- 为页面新增轻量辅助组件。
- 对现有组件做不破坏 props 和语义的小范围样式改造。

禁止：

- 为了视觉效果破坏业务语义。
- 修改 `front/src/types/api.ts`。
- 修改后端契约或 OpenAPI。
- 创建 mock、fixture、假业务数据。
- 伪造用户、单词、文章、题目、信件、统计结果。
- 新增依赖，除非单独说明必要性并获得确认。
- 把工程阶段说明展示给用户。

## 数据要求

不得用假数据填充页面。

如果没有真实数据或接口尚未接通，请设计真实产品状态：

- 加载态
- 空态
- 错误态
- 禁用态
- 可恢复操作
- 用户下一步

空态不能是“建设中”或“待生成”，而应该是用户能理解的产品文案，例如：

- 今天还没有学习任务
- 暂时无法加载今日任务
- 重新加载
- 返回今日学习
- 开始复习
- 继续阅读
- 写一封信
- 查看今日成果

## 禁止出现在 UI 中的内容

用户界面中禁止出现：

- 第一轮、第二轮、第三轮
- Round
- M1、M2、M3、M3C
- Web Foundation
- AI Studio
- 页面生成
- 组件货架
- 契约冻结
- OpenAPI
- DTO
- 壳路由
- 空壳
- Demo
- 建设中
- 待接入
- 生成前准备
- 业务页面未生成
- 只搭框架，不做业务页
- 供后续生成
- 这里仅保留

这些只能作为内部工程约束，绝不能成为用户可见文案。

## 输出分两步

第一步先不要写代码，请输出：

1. 你审计到的现有组件和样式资产。
2. 哪些组件计划按需复用，原因是什么。
3. 哪些组件只复用功能但需要重新设计视觉。
4. 哪些页面区域会新写页面级结构。
5. 计划新增或调整哪些 styles/tokens。
6. 五个页面的视觉方向和信息架构。
7. 如何避免 mock 数据和工程穿帮文案。

确认后，再进入第二步生成代码。

第二步生成代码时，只允许修改：

- `front/src/pages/*`
- `front/src/App.tsx`
- `front/src/components/layout/*`
- `front/src/styles/*`
- 必要的页面级辅助组件
- 必要的页面级 hooks 或 API client

不要修改：

- `front/src/types/api.ts`
- `backend/*`
- `backend/openapi/*`
- `package.json`
- 任何 mock、fixture、data 文件

最终结果应能通过：

```bash
cd front
npm run verify
```
