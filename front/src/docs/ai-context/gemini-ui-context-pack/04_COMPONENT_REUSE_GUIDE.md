# Component Reuse Guide

请先审计 `front/src/components`，再决定按需复用哪些组件。

复用判断标准：

应复用：

- 已封装交互行为的组件。
- 已封装表单语义的组件。
- 已封装加载、错误、空态、保存状态的组件。
- 已封装业务语义的组件，例如练习题、阅读、写信、安全检查、成果摘要。

可以新写：

- 页面级布局结构。
- 页面级视觉容器。
- 页面级装饰和分区。
- 页面级组合组件。
- 不涉及业务数据契约的小型展示组件。

复用组件时允许：

- 传入 `className`。
- 外层包裹新的视觉结构。
- 调整组件样式 class。
- 扩展视觉 token。
- 改造非业务逻辑的视觉结构。

复用组件时禁止：

- 改变 props 含义。
- 改变数据结构。
- 改变提交、保存、安全检查、反馈语义。
- 为了视觉效果绕开现有可访问性。
