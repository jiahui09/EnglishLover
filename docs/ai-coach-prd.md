# EnglishLover AI Coach PRD

> 文档定位：目标产品方案 / PRD 草案
> 范围边界：仅描述产品目标、用户体验、功能范围、验收标准和非目标；不涉及技术架构、数据库、接口或部署方案。
> 来源边界：基于用户本轮需求、EnglishLover 既有“今日学习 / 单词 / 阅读 / 结果反馈”产品方向，以及 WordForge 类项目中“轻量背词、卡片学习、SM-2、AI 阅读、学习统计”的可借鉴能力整理。
> UI 细化参考：`docs/ai-coach-ui-design-guidelines.md`；全局设计决策源：`DESIGN.md`。

## 1. 产品概述

### 1.1 产品名称

**EnglishLover AI Coach**

### 1.2 一句话定位

EnglishLover AI Coach 是一个面向考研英语学习者的个性化英语学习私教平台，通过“近期单词 -> AI 考研阅读 -> 练习题 -> 能力分析 -> 教练建议”的闭环，帮助用户把背过的单词转化为阅读理解能力。

### 1.3 核心判断

首版不应只做“背单词工具”，而应做一个**单词驱动阅读输出、阅读反哺词汇巩固、学习数据驱动私人教练建议**的英语学习系统。

### 1.4 产品价值

| 用户问题 | 产品解法 |
|---|---|
| 背了单词但不会在阅读中识别和使用 | AI 根据近期单词生成考研级阅读材料，让单词进入真实语境 |
| 做阅读题没有针对性 | 阅读题围绕用户近期单词、薄弱词和题型弱项生成 |
| 学习记录只是数字，没有指导意义 | 系统把词汇、阅读、题型、连续学习表现转化为能力分析和下一步建议 |
| 自学缺少陪伴和持续反馈 | 通过每日任务、阶段反馈、教练建议持续跟进学习进展 |

## 2. 目标用户

### 2.1 主用户

准备考研英语、需要提升词汇与阅读理解能力的中文学习者。

### 2.2 次用户

- 四六级后期希望向考研阅读难度过渡的学习者；
- 英语基础中等、想通过阅读巩固单词的自学者；
- 想要每日计划和学习反馈，但不想使用复杂课程系统的用户。

### 2.3 典型用户画像

| 画像 | 背景 | 痛点 | 期望 |
|---|---|---|---|
| 考研备考者 A | 每天背词，但阅读正确率不稳定 | 单词和阅读训练割裂 | 希望知道今天该背什么、读什么、错在哪里 |
| 自学提升者 B | 有一定词汇量，但缺乏输出训练 | 做题随机，无法形成体系 | 希望用最近学过的词生成针对性阅读 |
| 时间有限用户 C | 每天只有 15—30 分钟学习 | 不想自己规划材料 | 希望打开首页就能开始学习 |

## 3. 产品目标

### 3.1 首版目标

在首版中验证以下学习闭环是否成立：

```text
用户背诵近期单词
  -> 系统识别近期词与薄弱词
  -> AI 生成考研英语级阅读理解
  -> 用户完成阅读题
  -> 系统分析词汇、阅读、题型表现
  -> 给出下一步学习建议
```

### 3.2 成功标准

首版成功不以功能数量衡量，而以闭环是否能稳定完成衡量。

| 指标 | 首版目标 |
|---|---|
| 学习闭环完成 | 用户能完成“背词 -> 阅读 -> 做题 -> 看分析 -> 获得建议” |
| 任务清晰度 | 用户进入首页后能在 10 秒内知道今天该做什么 |
| 专注模式有效 | 用户开始单词背诵或阅读理解后，页面隐藏无关入口并只保留当前任务主行动 |
| 阅读相关性 | 阅读材料明确使用或围绕用户近期学习词 |
| 反馈可行动 | 学习建议必须指向下一步动作，而不是只展示统计数字 |
| AI 降级可用 | AI 不可用时，用户仍能完成基础学习任务 |

## 4. 核心范围

### 4.1 P0 首版必须做

| 模块 | 功能 | 说明 |
|---|---|---|
| 今日学习 | 今日任务清单 | 展示今日单词任务、阅读任务、教练建议入口 |
| 单词训练 | 单词背诵与复习 | 用户学习新词、复习近期词、标记掌握状态；开始后进入专注模式 |
| 阅读训练 | AI 考研阅读生成 | 根据近期单词生成阅读文章和练习题；阅读正文、答题和解析保持专注模式 |
| 阅读练习 | 题目作答与解析 | 支持主旨、细节、推断、词义题等基础题型 |
| 学习记录 | 学习数据记录 | 记录单词表现、阅读正确率、题型弱项、完成情况 |
| 教练建议 | 个性化学习指导 | 输出今日建议、弱项提醒、下一步学习路径 |

### 4.2 P1 应该做

| 模块 | 功能 | 说明 |
|---|---|---|
| 错词回流 | 阅读中错词进入复习队列 | 把阅读暴露出的薄弱词带回单词训练 |
| 难度调节 | 阅读难度建议 | 根据正确率建议保持、降低或提高难度 |
| 周学习报告 | 7 天学习总结 | 展示趋势、薄弱题型、词汇掌握变化 |
| 教练语气优化 | 鼓励式反馈 | 避免焦虑化、惩罚化文案 |

### 4.3 P2 后续再做

| 功能 | 延后原因 |
|---|---|
| 真实笔友系统 | 与首版“单词 + 阅读”闭环无直接关系，且涉及安全和内容治理 |
| 社区、排行榜 | 容易分散首版学习闭环 |
| 复杂成就系统 | 可以增强留存，但不是首版核心 |
| 重型管理后台 | 首版以学习体验验证为主 |
| 多阶段完整课程体系 | 容易过度扩张，先用每日闭环验证产品价值 |

## 5. 核心用户流程

### 5.1 每日学习主流程

```text
进入首页
  -> 查看今日学习任务
  -> 开始单词训练并进入单词专注模式
  -> 完成近期单词背诵/复习
  -> 生成或进入今日阅读并进入阅读专注模式
  -> 阅读文章并完成题目
  -> 查看解析与错因
  -> 查看教练建议
  -> 完成今日学习
```

### 5.2 单词到阅读的转化流程

```text
用户完成一组近期单词
  -> 系统提取今日新词、近期错词、重点词
  -> AI 生成考研英语阅读文章
  -> 文章自然融入目标词
  -> 题目检测用户是否真正理解这些词和文章逻辑
```

### 5.3 阅读到复习的回流流程

```text
用户提交阅读题
  -> 系统识别错误题型和错因
  -> 标记文章中的薄弱词
  -> 将薄弱词加入后续复习
  -> 教练给出下一步建议
```

## 6. 功能需求

### 6.1 今日学习

#### 用户故事

作为用户，我希望打开平台后直接看到今日学习安排，这样我不需要自己规划学习内容。

#### 功能要求

- 展示今日任务总览；
- 至少包含单词任务和阅读任务；
- 展示今日教练建议摘要；
- 支持从首页直接进入单词训练或阅读训练；
- 显示今日完成状态。

#### 验收标准

```gherkin
Given 用户进入今日学习页
When 今日存在未完成任务
Then 页面展示单词任务、阅读任务和教练建议入口

Given 用户完成单词任务
When 返回今日学习页
Then 单词任务状态显示为已完成或部分完成
```

### 6.2 单词训练

#### 用户故事

作为用户，我希望用简单卡片完成单词背诵，并让系统知道哪些词我掌握不好。

#### 功能要求

- 开始训练后进入单词专注模式，隐藏底部导航、学习画像、历史统计和无关推荐；
- 展示单词、释义、例句和掌握反馈；
- 支持“认识 / 模糊 / 不认识”或等价反馈；
- 记录近期学习词和薄弱词；
- 完成后提示这些词将用于阅读训练；
- 支持查看本轮结果。

#### 验收标准

```gherkin
Given 用户开始单词训练
When 用户完成一组单词反馈
Then 系统记录本轮掌握状态
And 页面提示可使用这些词生成阅读训练

Given 用户开始单词训练
When 训练进行中
Then 页面处于单词专注模式
And 仅保留返回、轻量进度、当前词和掌握反馈主行动
```

### 6.3 阅读训练

#### 用户故事

作为用户，我希望阅读文章能围绕我刚背过的词生成，这样背词和阅读训练能互相强化。

#### 功能要求

- 阅读文章难度对标考研英语；
- 阅读生成成功、使用预置阅读、阅读正文、答题和解析状态均进入阅读专注模式；
- 文章应自然包含近期单词或薄弱词；
- 每篇文章配套练习题；
- 题型至少覆盖主旨、细节、推断、词义四类中的若干类；
- 提供答案解析；
- 提供文章中目标词标记或回顾。

#### 验收标准

```gherkin
Given 用户近期已学习一组单词
When 用户生成阅读训练
Then 阅读任务说明中展示本次使用的目标词
And 阅读题提交后展示答案、解析和错因提示

Given 用户进入阅读文章或答题
When 阅读理解进行中
Then 页面处于阅读专注模式
And 仅保留返回、轻量进度、正文或题目、当前主行动
```

### 6.4 个性化统计

#### 用户故事

作为用户，我希望看到自己的学习表现，而不仅是完成了多少任务。

#### 功能要求

- 展示今日和近期学习记录；
- 展示词汇掌握情况；
- 展示阅读正确率；
- 展示题型弱项；
- 展示近期趋势；
- 不用复杂图表堆叠，优先让用户知道“下一步怎么做”。

#### 验收标准

```gherkin
Given 用户完成至少一次单词训练和阅读训练
When 用户进入学习画像或教练页
Then 页面展示词汇、阅读和题型表现摘要
And 至少给出一条可执行学习建议
```

### 6.5 教练建议

#### 用户故事

作为用户，我希望平台像私人教练一样告诉我下一步怎么学，而不是只展示数据。

#### 功能要求

- 输出今日建议；
- 输出弱项分析；
- 输出下一次学习动作；
- 使用鼓励式、具体、可执行的文案；
- 建议必须基于用户近期学习表现或明确标注为通用建议。

#### 验收标准

```gherkin
Given 用户有近期学习记录
When 系统生成教练建议
Then 建议包含学习原因和下一步行动
And 不只显示笼统鼓励语
```

## 7. 非目标

首版明确不做：

- 不做完整课程售卖系统；
- 不做开放社区；
- 不做真实笔友匹配；
- 不做排行榜；
- 不做复杂后台；
- 不做高复杂度学习能力雷达；
- 不做纯聊天式 AI 助教；
- 不把 AI 生成结果写成不可质疑的标准答案。

## 8. 体验原则

| 原则 | 说明 |
|---|---|
| 今日优先 | 用户首先看到今天该做什么 |
| 词文联动 | 单词训练必须服务阅读理解 |
| 专注模式 | 开始背词和阅读理解后隐藏无关入口，避免用户在学习中被统计、导航和推荐打断 |
| 输出强化 | 阅读题是对单词掌握的语境化检验 |
| 建议可执行 | 教练建议必须能转化为下一步动作 |
| 低压力 | 错误反馈应帮助用户修正，而不是制造焦虑 |
| AI 可降级 | AI 不可用时，学习闭环仍应可继续 |

## 9. 内容要求

### 9.1 单词内容

首版建议聚焦考研英语核心词，不追求一开始覆盖完整大词库。

每个单词建议包含：

- 英文词；
- 中文释义；
- 词性；
- 简短例句；
- 考研语境提示；
- 掌握状态。

### 9.2 阅读内容

每篇阅读建议包含：

- 英文标题；
- 英文正文；
- 考研难度说明；
- 融入的目标词；
- 练习题；
- 答案；
- 解析；
- 错因提示；
- 词汇回顾。

### 9.3 教练内容

教练建议必须包含：

- 当前表现；
- 主要问题；
- 原因解释；
- 下一步行动；
- 鼓励反馈。

示例：

> 你今天的细节题表现不错，但推断题出现 2 次错误。建议明天继续保持 10 个词复习量，并额外做 1 篇推断题较多的阅读。今天阅读中 `context` 和 `infer` 两个词建议加入明日复习。

## 10. 风险与控制

| 风险 | 控制方式 |
|---|---|
| AI 文章质量不稳定 | 提供人工可读的题目说明、解析检查入口和重生成入口 |
| 用户只背词不做阅读 | 首页把阅读作为单词完成后的下一步任务 |
| 用户被统计面板吓退 | 优先展示建议，不堆复杂图表 |
| 教练建议空泛 | 每条建议必须绑定一个具体学习行为 |
| 范围过大 | 首版只围绕“单词 + 阅读 + 练习题 + 教练建议” |

## 11. 首版验收清单

- [ ] 用户能完成一组单词学习；
- [ ] 系统能识别近期单词或薄弱词；
- [ ] 用户能基于近期单词进入阅读训练；
- [ ] 阅读训练包含考研级文章和练习题；
- [ ] 用户提交题目后能看到答案和解析；
- [ ] 错误题和薄弱词能进入学习反馈；
- [ ] 教练页能输出至少一条个性化建议；
- [ ] 首页能展示今日学习状态；
- [ ] AI 不可用时存在可解释的降级路径；
- [ ] 所有文案不把未验证学习效果写成确定承诺。
