# EnglishLover AI Coach UI 设计优化与细化方案

> 文档定位：目标产品方案 / UI 设计执行稿
> 设计要求：信息高效、画面直观简洁、提高专注力。
> 适用范围：今日学习、单词训练、阅读训练、教练建议；学习画像首版并入教练页摘要。
> 关联文档：`DESIGN.md`、`docs/ai-coach-prd.md`、`docs/ai-coach-information-architecture.md`、`docs/ai-coach-core-page-copy-and-interactions.md`。
> 范围边界：描述 UI 层级、视觉语言、组件规则、状态和验收；不描述后端接口、数据库或部署。

---

## 1. 设计目标

EnglishLover AI Coach 的首版界面不是数据看板，也不是课程商城，而是一个每天把用户带入学习闭环的私人教练。界面要让用户少判断、少选择、少被打断。

### 1.1 三条设计准则

| 准则 | 执行要求 | 避免 |
|---|---|---|
| 信息高效 | 每个模块只展示能支持下一步行动的信息 | 大段解释、重复统计、泛泛鼓励 |
| 画面直观简洁 | 用清晰层级、留白、卡片和状态标签表达结构 | 重型图表、炫彩渐变、装饰性插画 |
| 提高专注力 | 学习流程中每屏只保留一个主行动 | 多个同级按钮、弹窗打断、无关入口 |

### 1.2 首版成功体验

用户打开首页后应在 10 秒内知道：

```text
今天先做什么 -> 为什么做 -> 点哪里开始
```

用户完成一个页面后应立刻知道：

```text
刚完成了什么 -> 暴露了什么问题 -> 下一步做什么
```

## 2. 视觉方向

### 2.1 设计关键词

- 安静学习桌；
- 私人教练便签；
- 单词进入阅读语境的闭环轨迹；
- 低干扰、低焦虑、明确下一步。

### 2.2 色彩 token

| Token | Hex | 用途 |
|---|---:|---|
| Ink | `#17211B` | 标题、正文、关键文本 |
| Muted Ink | `#5F6B62` | 辅助说明、时间、次级标签 |
| Paper | `#F7F5EF` | 全局背景，模拟纸面但不过度复古 |
| Card | `#FFFFFF` | 卡片、阅读正文、输入区域 |
| Coach Green | `#2F6B4F` | 唯一主强调色：主按钮、当前进度、完成状态 |
| Word Amber | `#C7832B` | 薄弱词、待复习、注意提示 |
| Soft Red | `#B85C50` | 错误和生成失败，不用于惩罚性文案 |
| Line | `#E6E0D3` | 分割线、边框、禁用边界 |

使用限制：

- 每个页面的主色只能是 Coach Green。
- Word Amber 只用于学习风险或薄弱词，不用于装饰。
- 不使用彩虹渐变、霓虹色或营销页式大面积高饱和背景。

### 2.3 字体与排版

当前 Flutter 项目未配置自定义字体，因此首版使用系统字体，并通过字号、字重和间距建立秩序。

| 层级 | 建议尺寸 | 用法 |
|---|---:|---|
| 页面标题 | 24–28sp / 600–700 | 首页、训练页、教练页标题 |
| 卡片标题 | 18–20sp / 600 | 任务卡、结果卡、建议卡标题 |
| 正文 | 16sp / 400 | 普通说明、题目、解析 |
| 阅读正文 | 17–18sp / 400，行高 1.55–1.7 | 阅读文章，优先可读性 |
| 标签/数据 | 12–14sp / 500 | 进度、状态、题型标签 |

文案排版规则：

- 标题不超过 16 个汉字；
- 卡片说明控制在 1–2 行；
- 教练建议使用“判断 / 原因 / 下一步”三段，不写成长报告；
- 阅读正文段落之间保留明显间距。

### 2.4 空间与形状

| 项 | 规则 |
|---|---|
| 页面左右边距 | 移动端 20px；桌面端内容列最大宽度 720–840px |
| 卡片内边距 | 16–20px |
| 卡片间距 | 12–16px |
| 圆角 | 卡片 18–24px；按钮 14–16px |
| 阴影 | 极轻或不用；优先边框和背景差 |
| 分割线 | 只在信息层级需要时使用 |

### 2.5 签名元素：学习闭环轨迹

为避免通用模板感，EnglishLover 的标志性 UI 元素是“学习闭环轨迹”：

```text
单词 -> 阅读 -> 解析 -> 建议
```

呈现规则：

- 首页可用横向或纵向四步短路径显示今日闭环进度；
- 结果页用当前完成节点高亮下一节点；
- 不做复杂动画，最多用轻微状态过渡；
- 轨迹是结构提示，不是装饰图案。

## 3. 页面级设计

### 3.1 今日学习：首页

#### 页面目标

回答“我今天该怎么学？”，并把用户带入最短学习路径。

#### 信息优先级

1. 今日一句话计划；
2. 当前最推荐任务；
3. 单词/阅读两个任务状态；
4. 简短教练提示；
5. 今日完成进度。

#### 推荐结构

```text
今日学习
├── 顶部：问候 + 今日一句话计划
├── 闭环轨迹：单词 -> 阅读 -> 解析 -> 建议
├── 主任务卡：当前最该做的一步
├── 次级任务卡：单词 / 阅读状态摘要
├── 教练便签：一句判断 + 一个入口
└── 底部导航：今日 / 单词 / 阅读 / 教练
```

#### 首屏示例

```text
今天先把近期词放进阅读里
15–25 分钟 · 单词 20 个 · 阅读 1 篇

[闭环轨迹：单词 进行中 -> 阅读 -> 解析 -> 建议]

当前建议
先完成一组单词训练，阅读会更贴合你今天的词。

[开始单词训练]
```

#### 设计规则

- 首页首屏只放一个主按钮；
- 单词和阅读任务卡可以并列显示状态，但不能和主任务抢主按钮；
- 教练建议摘要最多 2 行；
- 完成状态优先显示“明天优先复习什么”，而不是庆祝动效。

#### 状态细化

| 状态 | 主卡内容 | 主行动 |
|---|---|---|
| 无记录 | 先学习 5 个考研核心词，生成第一篇阅读 | 开始第一组单词 |
| 单词未完成 | 完成近期词复习，准备生成阅读 | 开始单词训练 |
| 单词完成 / 阅读未做 | 用刚复习的词生成阅读 | 生成今日阅读 |
| 阅读完成 / 未看建议 | 推断题或薄弱词摘要 | 看教练建议 |
| 今日完成 | 今日闭环完成，明天优先复习薄弱词 | 查看今日总结 |
| AI 不可用 | 使用预置阅读继续完成今日任务 | 使用预置阅读 |

### 3.2 单词训练页

#### 页面目标

让用户快速完成知识输入，并留下真实掌握反馈。

#### 信息优先级

1. 当前词；
2. 查看释义或掌握反馈；
3. 例句和考研语境提示；
4. 本轮进度；
5. 退出或稍后继续。

#### 推荐结构

```text
单词训练
├── 顶部轻量进度：第 6 / 20 个
├── 单词卡片：英文词 + 提示
├── 展开区：释义 + 例句 + 考研语境
├── 三态反馈：认识 / 模糊 / 不认识
└── 底部提示：这些词会用于稍后的阅读
```

#### 设计规则

- 训练中隐藏复杂导航，只保留返回和进度；
- 单词卡片居中但不空洞，保证释义展开后仍能一屏理解；
- “认识 / 模糊 / 不认识”视觉权重一致，避免诱导用户选“认识”；
- 薄弱词反馈不使用责备语气，只提示后续会回炉。

#### 本轮结果页

```text
本轮完成
认识 14 · 模糊 4 · 不认识 2

建议先用这些词做一篇阅读，看看它们在文章里是否稳定。

[生成阅读]
[稍后再说]
```

结果页主行动是“生成阅读”；“稍后再说”为文本按钮。

---

### 3.3 阅读训练页

#### 页面目标

把近期单词放入考研阅读语境，并通过题目检测理解。

#### 信息优先级

1. 阅读来源与目标词；
2. 文章正文；
3. 题目；
4. 答案解析；
5. 错词回流和教练建议。

#### 四个模式

| 模式 | 页面重点 | 主行动 |
|---|---|---|
| 生成前 | 说明使用哪些近期词、预计题型和用时 | 生成今日阅读 |
| 阅读中 | 正文可读性、目标词轻标记、少干扰 | 开始答题 |
| 答题中 | 题目、选项、作答进度 | 提交答案 |
| 解析中 | 正确率、错因、原文依据、下一步 | 看教练建议 |

#### 阅读模式设计规则

- 阅读正文使用白色或近白卡片，行高 1.55–1.7；
- 目标词标记用浅底色或下划线，不打断阅读；
- 词汇提示默认折叠，用户点击目标词再查看；
- 答题入口固定在正文之后或底部，不悬浮遮挡正文；
- 解析必须给出原文依据，不只显示对错。

#### AI 失败降级

```text
暂时无法生成新文章
可以先使用预置考研阅读继续练习，本次建议会标注为“基于预置阅读”。

[使用预置阅读]
[重试生成]
```

降级规则：

- 不清空已选择的目标词；
- 不假装 AI 已成功生成；
- 预置阅读完成后仍进入解析和教练建议。

---

### 3.4 教练建议页

#### 页面目标

把学习记录转化为具体下一步，而不是展示复杂统计。

#### 信息优先级

1. 今日一句话建议；
2. 当前表现判断；
3. 主要原因；
4. 下一步任务；
5. 可折叠学习画像。

#### 推荐结构

```text
教练建议
├── 今日建议：一句话
├── 当前判断：词汇 / 阅读 / 题型
├── 原因解释：不超过 3 点
├── 下一步任务：复习词 + 阅读题型 + 复盘重点
├── 学习画像摘要：可展开
└── 主行动：开始推荐任务
```

#### 设计规则

- 教练页不使用仪表盘首屏；
- 所有数据后面必须连接行动；
- 数据不足时给入门建议，不生成伪个性化结论；
- 鼓励反馈放在底部，不能替代判断和行动。

#### 示例

```text
今日建议
先复习 4 个薄弱词，再做一篇包含推断题的阅读。

当前判断
词汇识别稳定，但在长句和转折语境里不够稳。

下一步
1. 复习 context、assume、maintain、evaluate
2. 做 1 篇推断题强化阅读
3. 复盘 however / although 后面的作者态度

[开始薄弱词复习]
```

## 4. 组件规则

### 4.1 主行动按钮

- 每个页面同一屏最多一个主行动；
- 使用 Coach Green；
- 文案必须是动词短语：开始单词训练、生成今日阅读、提交答案、看教练建议；
- 加载时保留按钮位置，文案改为进行中状态。

### 4.2 任务卡

任务卡用于首页和结果页，结构固定：

```text
[状态标签]
任务名称
一句说明
关键数据（最多 3 个）
[主行动或次行动]
```

禁止在任务卡内堆叠长列表、图表或两个以上同级按钮。

### 4.3 教练便签卡

用于短建议，结构固定：

```text
教练建议
一句判断。
下一步：一个明确动作。
```

便签卡不是聊天气泡，不提供自由聊天入口。

### 4.4 进度提示

- 学习中使用轻量进度条或文字：`第 6 / 20 个`；
- 首页使用闭环轨迹表达整体进度；
- 不使用连续打卡火焰、排行榜或强刺激进度系统。

### 4.5 状态反馈

| 状态 | UI 规则 | 文案规则 |
|---|---|---|
| Loading | 骨架屏或轻量进度，不遮盖已存在内容 | 说明正在做什么 |
| Empty | 简洁空卡 + 一个主行动 | 告诉用户第一步 |
| Error | 错误卡 + 重试 + 降级 | 说明影响和恢复路径 |
| Success | 结果摘要 + 下一步 | 不过度庆祝 |
| Disabled | 降低对比 + 条件说明 | 说明如何解锁 |

## 5. 专注模式与专注力约束

专注模式是单词背诵和阅读理解的学习中状态，不是新的一级页面。用户从首页、单词页或教练页点击开始训练后，界面进入专注模式；完成当前轮次、主动返回或确认退出后离开专注模式。

### 5.1 专注模式通用外壳

```text
专注模式
├── 顶部：返回 / 当前任务标题 / 轻量进度
├── 主内容：当前词或当前阅读材料
├── 当前任务反馈：掌握反馈、答题选项或解析
└── 底部：唯一主行动
```

通用规则：

- 隐藏底部导航、学习画像、历史统计、设置入口和非当前任务推荐；
- 顶部只保留返回、任务标题和轻量进度，不放入口型导航；
- 同屏最多一个主行动，次行动降为文本或折叠到结果页；
- 退出未完成训练时才出现确认弹窗，避免误触丢失进度；
- 完成当前轮次后可退出专注模式，或切换到下一段专注任务。

### 5.2 单词背诵专注模式

触发：用户点击“开始单词训练”“开始第一组单词”“开始薄弱词复习”后进入。

```text
单词专注模式
├── 顶部：第 6 / 20 个 · 本轮单词
├── 当前词：significant
├── 释义揭示：查看释义 / 收起释义
├── 语境提示：例句 + 考研语境
└── 掌握反馈：认识 / 模糊 / 不认识
```

设计规则：

- 当前词和三态反馈是视觉中心；
- “认识 / 模糊 / 不认识”权重一致，避免诱导选择；
- 进度只显示当前位置和本轮总量，不展示复杂统计；
- 标记反馈后自动进入下一个词，不弹出额外确认；
- 完成本轮后展示结果摘要，并以“用这些词生成阅读”为主行动。

### 5.3 阅读理解专注模式

触发：用户点击“生成今日阅读”后进入生成前/生成中状态；文章生成成功、使用预置阅读、开始阅读正文或进入答题后保持阅读专注模式。

```text
阅读专注模式
├── 顶部：阅读进度 / 题目进度
├── 阅读正文：标题 + 正文 + 轻量目标词标记
├── 可选提示：目标词提示默认折叠
├── 答题区：题目、选项、当前题进度
└── 解析区：答案、原文依据、错因、下一步
```

设计规则：

- 阅读正文优先，生成说明、目标词和题目分阶段出现；
- 目标词提示默认折叠，点击目标词再查看释义或语境；
- 阅读中主行动是“开始答题”，答题中主行动是“提交答案”，解析中主行动是“看教练建议”；
- AI 失败时在当前专注模式内提供“使用预置阅读”和“重试生成”，不跳回首页；
- 解析必须给出原文依据和下一步，不只显示正确/错误。

### 5.4 专注模式中的隐藏项

在单词背诵、阅读正文、答题和解析中，默认隐藏或弱化：

- 底部导航；
- 学习画像入口；
- 设置入口；
- 历史记录入口；
- 复杂统计；
- 非当前任务的推荐；
- “再做一篇”等可能抢占当前闭环的行动。

### 5.5 弹窗使用限制

只允许以下场景使用弹窗：

- 用户退出未完成训练；
- AI 内容生成失败且需要选择重试或降级；
- 提交答案前确认未答题项。

其他提示使用页面内反馈，不打断学习。

### 5.6 次级行动排序

当页面需要多个行动时，权重固定：

```text
主按钮：继续当前闭环
次按钮：查看解释 / 稍后再说
文本按钮：返回 / 更换 / 跳过
```

不能让“再做一篇”“查看统计”“更换文章”与主闭环行动同权重。

## 6. 响应式设计

### 6.1 移动端优先

- 底部导航：今日 / 单词 / 阅读 / 教练；普通页面显示，专注模式隐藏；
- 单列卡片；
- 主行动靠近内容结尾，学习流程可使用底部固定按钮；
- 阅读正文宽度跟随屏幕，保持舒适行高。

### 6.2 桌面端延展

- 内容居中，最大宽度 720–840px；
- 阅读页可使用双栏：正文在左，目标词/题目进度在右；
- 首页可以把任务卡横向排列，但当前建议仍占主视觉；
- 不因屏幕变宽而增加额外入口。

## 7. 可实现性约束

当前代码事实：Flutter 应用仍是默认计数器模板，依赖中没有图表、自定义字体或复杂动画库。因此首版设计应使用 Flutter Material 基础能力实现。

### 7.1 不新增依赖的默认方案

| 需求 | 默认实现方向 |
|---|---|
| 卡片 | `Card` / `Container` + `BoxDecoration` |
| 底部导航 | `NavigationBar` 或 `BottomNavigationBar` |
| 主按钮 | `FilledButton` / 自定义主题按钮 |
| 次按钮 | `OutlinedButton` / `TextButton` |
| 进度 | `LinearProgressIndicator` + 文本 |
| 标签 | 小型 `Container` + 文本 |
| 阅读正文 | `SingleChildScrollView` + 文本样式 |
| 状态反馈 | 页面内卡片，不默认弹窗 |

### 7.2 Token 落地建议

后续实现时应先建立主题常量，再写页面：

```text
颜色 -> 字号 -> 间距 -> 圆角 -> 组件变体 -> 页面
```

禁止在多个页面各自硬编码颜色和按钮样式。

## 8. UI 验收清单

### 8.1 首页

- [ ] 用户 10 秒内能看出今天先做什么；
- [ ] 首屏只有一个主按钮；
- [ ] 单词、阅读、教练建议的关系可见；
- [ ] 今日完成状态给出明日或下一步建议。

### 8.2 单词训练

- [ ] 进入单词训练后隐藏底部导航并显示专注模式外壳；
- [ ] 当前词和反馈按钮是视觉中心；
- [ ] 进度清楚但不抢焦点；
- [ ] 三个掌握反馈按钮权重一致；
- [ ] 结果页明确引导生成阅读。

### 8.3 阅读训练

- [ ] 进入阅读正文和答题后隐藏底部导航并显示专注模式外壳；
- [ ] 阅读正文行高、段距、对比度适合长时间阅读；
- [ ] 目标词标记不破坏阅读；
- [ ] 答题和解析状态清楚分离；
- [ ] AI 失败时可使用预置阅读继续闭环。

### 8.4 教练建议

- [ ] 建议由“判断 / 原因 / 下一步”组成；
- [ ] 数据不足时不伪装个性化；
- [ ] 学习画像是摘要或折叠内容；
- [ ] 主行动指向下一次学习任务。

### 8.5 全局

- [ ] 所有错误、空状态、成功状态都有下一步；
- [ ] 没有排行榜、商城、社区等非首版入口；
- [ ] 没有超过一个同屏主 CTA；
- [ ] 单词背诵和阅读理解开始后均进入专注模式；
- [ ] 颜色、间距、圆角、字体层级可沉淀为主题 token。
