## 1. 信息架构总览

EnglishLover AI Coach 首版采用“今日学习驱动”的结构。用户不需要先进入复杂功能菜单，而是从首页直接进入每天的学习闭环。

```text
EnglishLover AI Coach
├── 今日学习
│   ├── 今日任务总览
│   ├── 单词任务卡
│   ├── 阅读任务卡
│   ├── 教练建议摘要
│   └── 今日完成进度
├── 单词训练
│   ├── 今日新词
│   ├── 到期复习
│   ├── 薄弱词回炉
│   ├── 单词专注模式
│   ├── 单词卡片训练
│   └── 本轮单词结果
├── 阅读训练
│   ├── 生成阅读
│   ├── 阅读专注模式
│   ├── 阅读文章
│   ├── 目标词提示
│   ├── 练习题
│   ├── 答案解析
│   └── 阅读结果
├── 学习画像
│   ├── 词汇掌握
│   ├── 阅读表现
│   ├── 题型弱项
│   ├── 近期趋势
│   └── 学习连续性
└── 教练建议
    ├── 今日建议
    ├── 弱项分析
    ├── 下一步任务
    ├── 复习提醒
    └── 阶段总结
```

## 2. 一级导航

| 导航项 | 页面目标 | 首版优先级 |
|---|---|---|
| 今日学习 | 告诉用户今天该学什么，并承接所有主流程入口 | P0 |
| 单词训练 | 完成单词输入和复习，开始后进入单词专注模式 | P0 |
| 阅读训练 | 基于近期词完成考研英语阅读输出和题目练习，阅读/答题时进入阅读专注模式 | P0 |
| 学习画像 | 汇总用户词汇、阅读、题型表现 | P1 |
| 教练建议 | 给出具体学习指导和下一步行动 | P0 |

首版可以将“学习画像”和“教练建议”合并显示，但信息架构上应区分：

- 学习画像回答：**我现在怎么样？**
- 教练建议回答：**我下一步该怎么做？**

## 4. 首版主路径

### 4.1 标准每日路径

```text
今日学习
  -> 单词专注模式
  -> 本轮单词结果
  -> 阅读专注模式
  -> 阅读结果
  -> 教练建议
  -> 今日学习完成
```

### 4.2 用户只想背词

```text
今日学习
  -> 单词专注模式
  -> 本轮单词结果
  -> 提示“用这些词做一篇阅读”
  -> 用户可暂时跳过
```

设计要求：

- 可以跳过阅读，但页面应清楚提示：阅读是巩固这些单词的推荐下一步。

### 4.3 用户直接做阅读

```text
今日学习
  -> 阅读训练
  -> 若近期单词不足，提示先完成单词任务
  -> 进入阅读正文或答题后切换为阅读专注模式
  -> 或使用系统推荐核心词生成阅读
```

设计要求：

- 不阻断用户；
- 但应说明“先背词再阅读”的学习收益。

### 4.4 AI 不可用路径

```text
今日学习
  -> 阅读训练
  -> AI 生成失败
  -> 使用预置考研阅读练习
  -> 完成题目和解析
  -> 教练建议标注“本次基于预置阅读生成”
```

设计要求：

- AI 失败不能让今日学习中断；
- 降级选择保留在阅读专注模式中，不跳回首页；
- 降级文案要透明，不假装 AI 成功。

## 6. 状态设计

### 6.1 空状态

| 页面 | 空状态 | 行动 |
|---|---|---|
| 今日学习 | 还没有学习记录 | 开始第一组单词 |
| 单词训练 | 暂无近期词 | 选择考研核心词开始 |
| 阅读训练 | 近期词不足 | 先背 5 个词，或使用系统推荐词 |
| 学习画像 | 数据不足 | 完成一次单词和阅读后生成画像 |
| 教练建议 | 暂无足够记录 | 给出通用入门建议 |

### 6.2 加载状态

| 场景 | 文案方向 |
|---|---|
| 生成阅读 | 正在把你的近期单词组织成考研阅读材料 |
| 分析结果 | 正在分析你的词汇和题型表现 |
| 生成建议 | 正在整理下一步学习建议 |

### 6.3 错误状态

| 场景 | 处理 |
|---|---|
| AI 生成失败 | 说明失败原因，提供重试和预置阅读 |
| 阅读题提交失败 | 保留已选答案，允许重试 |
| 学习记录不足 | 给出通用建议，不假装个性化 |
| 内容质量不满意 | 支持重新生成或更换文章 |

## 7. 移动端结构

移动端首版建议使用底部导航：

```text
今日 | 单词 | 阅读 | 教练
```

“学习画像”在移动端可作为“教练”页中的二级内容，避免底部导航过多。

移动端页面优先级：

1. 今日任务卡；
2. 当前训练内容；
3. 下一步行动按钮；
4. 简短反馈；
5. 完整统计折叠展示。

专注模式移动端规则：

- 开始单词背诵或阅读理解后隐藏底部导航；
- 顶部只保留返回、任务标题和轻量进度；
- 底部可固定当前唯一主行动，但不能遮挡单词卡、阅读正文或题目选项；
- 完成本轮或确认退出后恢复普通导航。

## 8. 信息架构非目标

首版不设置以下一级入口：

- 社区；
- 排行榜；
- 商城；
- 课程中心；
- 笔友广场；
- 管理后台；
- AI 聊天大厅。

原因：这些入口会稀释“单词 + 阅读 + 私教建议”的主闭环。
